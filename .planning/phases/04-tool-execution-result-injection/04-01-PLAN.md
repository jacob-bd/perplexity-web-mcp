Add `format_tool_results()` function to prompt_builder.py that formats tool results for injection into Perplexity prompts.

**Function signature:**
```python
def format_tool_results(
    tool_results: dict[str, tuple[str, bool]],
    max_results: int = 3,
    max_chars_per_result: int = 1000
) -> str
```

**Implementation requirements:**
1. Accept dict mapping tool_use_id -> (content, is_error) tuples
2. Return empty string if no tool results
3. Limit to most recent `max_results` results (last N items from dict)
4. Format results as structured markdown:
   - Section header: "TOOL RESULTS:"
   - Separator: "---"
   - For each result:
     - If error: "Tool: {tool_use_id} (ERROR)\nError: {content}"
     - If success: "Tool: {tool_use_id}\nResult: {content}"
   - Separator: "---" after each result
   - Blank line at end
5. Truncate individual result content to `max_chars_per_result` with "... (truncated)" marker
6. Maintain insertion order (Python 3.7+ dict guarantee)

**Example output format:**
```
TOOL RESULTS:
---
Tool: toolu_abc123
Result:
Found 3 Python tutorials covering basics and advanced topics.
---
Tool: toolu_def456 (ERROR)
Error: Network timeout after 30 seconds
---

```

**Why this format:** Structured markdown with clear labels ensures Perplexity models can parse and reference results. Error distinction prevents models from using failed tool results as if they succeeded.
