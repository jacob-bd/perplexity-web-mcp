Wrap the entire strategy chain in defensive exception handling:
```python
def parse_response(answer: str) -> ParseResult:
    """Parse response with defensive exception handling.

    Returns a parse result even if all strategies fail - confidence will be 0.0.
    """
    try:
        # ... existing strategy chain code ...
    except Exception as e:
        logger.error(f"Critical parsing failure: {type(e).__name__}: {e}", exc_info=True)
        # Return empty result rather than crashing
        return ParseResult(
            strategy="error",
            tool_calls=[],
            confidence=0.0
        )
```

This ensures parse_response ALWAYS returns a valid ParseResult, even on catastrophic failures.

DO NOT add this to individual strategies - they should raise normally. This is top-level defensive handling.

Why this approach: Defense-in-depth - even if strategies have bugs, parser won't crash. Server-level handler will still catch and fallback gracefully.
