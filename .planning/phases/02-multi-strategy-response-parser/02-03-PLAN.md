  <behavior>
    Extract function-like patterns from inline code

    Cases:
    - "I'll run `search('weather')` for you" -> [{'name': 'search', 'args_raw': "'weather'"}]
    - "Let me `read_file('test.txt')` now" -> [{'name': 'read_file', 'args_raw': "'test.txt'"}]
    - "Using `calculate(1, 2, 3)` to compute" -> [{'name': 'calculate', 'args_raw': '1, 2, 3'}]
    - "`variable` is not a function" -> []
    - "`search()` with no args" -> [{'name': 'search', 'args_raw': ''}]
    - "Multiple: `func1()` and `func2('arg')`" -> [{'name': 'func1', ...}, {'name': 'func2', ...}]
  </behavior>
  <implementation>
    1. Define INLINE_CODE_PATTERN: r'`([^`]+)`'
    2. Extract all backtick contents with re.findall
    3. For each extracted code, check if it matches function pattern
    4. Use FUNCTION_PATTERN: r'(\w+)\s*\(([^)]*)\)'
    5. Extract function name and raw arguments
    6. Return list of detected function mentions
    7. Ignore non-function code snippets
  </implementation>
</feature>
