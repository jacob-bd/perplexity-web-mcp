# Milestone v1.0: Best-Effort Tool Calling

**Status:** ✅ SHIPPED 2026-02-05
**Phases:** 1-5
**Total Plans:** 17

## Overview

Transform Perplexity web models into tool-executing agents through prompt engineering and flexible parsing. This roadmap delivers best-effort tool calling (40-60% reliability) by injecting tool definitions using training-data-familiar formats and extracting calls through multi-strategy parsing with graceful fallbacks.

## Milestone Summary

**Key Decisions:**

- Accept 40-60% reliability target (Research confirms fundamental training mismatch)
- Use Python in markdown as primary format (Most likely to trigger trained behavior)
- Build multi-strategy parser (Models will deviate from any single format)
- Use Google-style docstrings (More common in training data than numpy-style)
- Keep type annotations simple (Avoid complex generics that might confuse models)
- Position tools before user message (Ensures models see available tools before query)
- Support dual injection formats (Full Python primary, minimal bullet-point fallback)
- Use dual regex patterns for key-value extraction (Strict uppercase primary with case-insensitive fallback)
- Use ast.walk() for comprehensive extraction (Finds all calls regardless of control flow)
- Use regex-based markdown extraction (mdextractor package had installation issues)
- Apply strategies in confidence order (AST: 0.9, key-value: 0.7, inline: 0.5)
- Use confidence threshold of 0.7 for tool_use blocks (Inline_code strategy at 0.5 would return false positives)
- Generate tool_use IDs with toolu_{24 hex} format (Matches Anthropic API specification)
- Set stop_reason to tool_use when tool blocks present (Signals to clients that tool execution required)
- Stream complete tool input JSON in single event (Simpler than streaming chunks, Anthropic spec allows both)
- Parse response in producer thread for streaming (Producer has full text)
- Validate tool pairing before processing (Fail fast on invalid requests)
- Extract tool results early, inject later (Validation and logging in Phase 3, injection in Phase 4)
- Limit to 3 most recent results by default (Maintains focus, prevents context bloat)
- Truncate individual results to 1000 chars (Prevents context bloat with marker for clarity)
- Tool results injected after tool definitions, before user message (Ensures model sees execution results)
- Log confidence scores at INFO level when tool calls detected (TC-03 visibility)

**Issues Resolved:**

- Previous XML/ReAct formats failed completely (root cause: training objective mismatch)
- Models default to explaining rather than executing (addressed via multi-strategy parsing)
- No system prompt distinction in Perplexity (addressed via tool injection in user content)
- Format deviation unpredictable (addressed via fallback chain)

**Technical Debt Incurred:**

- Real-world testing with live Perplexity models needed (mocked during development)
- Success rate metrics collection required for production validation
- Potential adjustment of confidence threshold based on real-world performance
