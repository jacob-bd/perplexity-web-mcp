## 6. Potential Integration Risks

### 6.1 Real-World Perplexity Responses
**Risk Level:** MEDIUM (Acceptable for v1)

**Issue:** All tests use mocked Perplexity responses. Real models may return unexpected formats.

**Mitigation:**
- Multi-strategy parser with fallback chain
- Graceful degradation on parse failures
- Comprehensive logging for debugging
- Expected success rate: 40-60% (documented in requirements)

**Recommendation:** Manual testing with real Perplexity API before production use.

### 6.2 Streaming Mode Parsing
**Risk Level:** LOW

**Issue:** Streaming mode parses accumulated text at end (line 1269). Partial tool calls during streaming not detected.

**Verification:** `test_streaming_with_tool_results` confirms parsing happens after stream completes.

**Impact:** Acceptable - tool_use blocks appear in final response, not mid-stream.

### 6.3 Confidence Threshold Hardcoding
**Risk Level:** LOW

**Issue:** Confidence threshold 0.7 is hardcoded (line 774, 1082).

**Current State:** Works as designed, documented in requirements.

**Future:** Could be made configurable per request.
