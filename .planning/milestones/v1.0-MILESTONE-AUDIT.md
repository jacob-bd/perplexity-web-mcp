## End-to-End User Flows

### Flow 1: Basic Tool Calling ✅
1. User → POST /v1/messages with tools array
2. System → inject tool definitions (Phase 1)
3. Perplexity → response with tool-like content
4. Parser → extract tool calls (Phase 2)
5. API → return tool_use blocks (Phase 3)
6. User → send tool_result with execution
7. System → inject results into next prompt (Phase 4)
8. Conversation continues

**Verified:** `test_full_tool_calling_cycle` (tests/test_tool_result_flow.py:44)

### Flow 2: Graceful Degradation ✅
1. User → POST /v1/messages with tools array
2. System → inject tool definitions
3. Perplexity → unparseable response or low confidence
4. System → classify_parse_error (Phase 5)
5. API → return text-only response (stop_reason: end_turn)
6. Logs → internal error details at ERROR level
7. User → receives conversational response

**Verified:** `test_complete_graceful_degradation_flow` (tests/test_reliability.py:15)

### Flow 3: Observability ✅
1. Administrator → GET /metrics
2. System → return Prometheus exposition format
3. Metrics → parse_attempts_total, parse_confidence, tool_calls_detected, parse_duration
4. Monitoring → success/failure rates per strategy visible

**Verified:** `test_metrics_endpoint_exists` (tests/test_api_integration.py:695)

## Risk Assessment

### Medium Risk: Real Model Behavior
**Description:** Real Perplexity responses may differ from test mocks
**Impact:** Success rate may vary from expected 40-60%
**Mitigation:**
- Multi-strategy parser handles format variations
- Graceful degradation for unparseable responses
- Comprehensive metrics track actual performance
- System designed for best-effort, not guaranteed success

**Recommendation:** Manual testing with live Perplexity models before production deployment

### Low Risk: Streaming Parse Timing
**Description:** Streaming mode parses complete response, not mid-stream
**Impact:** Tool use blocks appear at end of stream
**Mitigation:** Design choice documented in Phase 3
**Status:** Acceptable as-is

### Low Risk: Hardcoded Threshold
**Description:** Confidence threshold fixed at 0.7
**Impact:** Cannot adjust without code change
**Mitigation:** Threshold chosen based on research and testing
**Status:** Acceptable for v1.0
